# Ответы на тестовые вопросы

1. b) Ошибка компиляции - нельзя сортировать константный вектор
   Объяснение: Константный вектор нельзя модифицировать, а sort изменяет порядок элементов.

2. c) `const int* const ptr = &value;`
   Объяснение: Это единственный вариант, где и указатель, и данные константны, и есть инициализация.

3. c) 1 2
   Объяснение: mutable позволяет изменять поле даже в константном методе.

4. b) Для кэширования результатов вычислений
   Объяснение: Это классический пример правильного использования mutable - кэширование не меняет логическое состояние объекта.

5. c) Вариант 3
   Объяснение: IILE используется для сложной инициализации с дополнительными действиями.

6. b) Ошибка компиляции - нельзя модифицировать поля в const методе
   Объяснение: Константный метод не может изменять поля класса без mutable.

7. b) Вариант 2
   Объяснение: Правильное объявление константного метода, возвращающего константную ссылку.

8. d) Вариант 4
   Объяснение: Только этот вариант корректен - можно перемещать константный итератор, но нельзя менять значение через него.

9. c) Ошибка компиляции в modify()
   Объяснение: Константная ссылка на вектор запрещает его модификацию, push_back вызовет ошибку компиляции.

10. c) Логическая константность гарантирует неизменность наблюдаемого состояния объекта
    Объяснение: Это основной принцип логической константности - сохранение видимого пользователю состояния.

11. a) Вариант 1
    Объяснение: Использование const_cast допустимо только для работы с legacy API, которое не поддерживает const.

12. c) Вариант 3
    Объяснение: Этот вариант корректно использует константный метод, мьютекс помечен как mutable, и возвращает константную ссылку.

13. d) Неопределенное поведение
    Объяснение: Модификация константного объекта через const_cast приводит к неопределенному поведению.

14. b) Вариант 2
    Объяснение: Для строк и других больших объектов правильно передавать по константной ссылке.

15. b) Ошибка компиляции - нельзя модифицировать поля в константном операторе
    Объяснение: Оператор присваивания не может быть константным, так как должен модифицировать объект.

16. c) Возвращаемое значение get() всегда константно
    Объяснение: const T& гарантирует константность возвращаемого значения независимо от типа T.

17. d) Утечка памяти
    Объяснение: Синглтон создается через new, но нет механизма освобождения памяти.

18. b) Вариант 2
    Объяснение: cbegin() и cend() правильно используются для константной итерации.

19. c) Ошибка компиляции - несоответствие константности
    Объяснение: Виртуальный метод в производном классе должен иметь ту же константность.

20. d) Вариант 4
    Объяснение: Правильное объявление константного указателя на константный метод.

21. c) Код скомпилируется успешно
    Объяснение: Поля помечены как mutable, что позволяет их модифицировать в константном методе.

22. a) Вариант 1
    Объяснение: Правильная реализация перегрузки для константных и неконстантных объектов.

23. d) Неопределенное поведение
    Объяснение: Модификация константного объекта через const_cast приводит к неопределенному поведению.

24. b) Вариант 2
    Объяснение: Вектор передается по константной ссылке для эффективности, а size по значению, так как это примитивный тип.

25. b) Ошибка компиляции - метод не помечен как const
    Объяснение: Переопределяющий метод должен иметь ту же константность, что и базовый.

26. b) Вариант 2
    Объяснение: Структурированное связывание с const auto& обеспечивает безопасный доступ только для чтения.

27. c) Ошибка компиляции - возвращаемое значение не должно быть константным
    Объяснение: Возвращаемое значение по значению не нужно делать константным.

28. b) const методы могут быть не потокобезопасны при использовании mutable
    Объяснение: mutable поля могут изменяться в константных методах, что может привести к гонкам данных.

29. c) Ошибка компиляции - нельзя копировать unique_ptr
    Объяснение: auto ptr = w.getPtr() пытается скопировать unique_ptr, что запрещено.

30. b) Вариант 2
    Объяснение: Правильное переопределение константного виртуального метода с override.

31. b) Ошибка компиляции - нельзя модифицировать поля в константном методе
    Объяснение: Константный метод не может изменять поля класса, даже в шаблонном методе.

32. b) Вариант 2
    Объяснение: Оператор присваивания должен возвращать неконстантную ссылку на объект.

33. a) Код скомпилируется успешно
    Объяснение: friend-функция может возвращать константное значение и принимать константные параметры.

34. d) Все вышеперечисленное верно
    Объяснение: Все утверждения корректно описывают поведение const с умными указателями.

35. a) Код скомпилируется успешно
    Объяснение: Правильное использование protected виртуального константного метода.

36. a) Вариант 1
    Объяснение: Правильная реализация перегрузок для lvalue и rvalue с учетом константности.

37. a) Код скомпилируется успешно
    Объяснение: Оператор приведения может быть константным и возвращать константную ссылку.

38. d) Вариант 4
    Объяснение: Правильное использование const для параметра и доступа только для чтения.

39. a) Код скомпилируется успешно
    Объяснение: Корректное переопределение константного виртуального метода и неконстантный метод установки значения.

40. d) Все лямбды корректны и имеют разное поведение
    Объяснение: Каждая лямбда демонстрирует различные аспекты использования const.
