# Битовые поля и упаковка данных

## Определение
Битовые поля - механизм C++ для точного контроля над размером полей структуры/класса, позволяющий упаковывать данные на уровне битов.

## Зачем это нужно
- Экономия памяти для структур с множеством маленьких полей
- Прямая работа с регистрами железа
- Совместимость с бинарными протоколами
- Эффективное хранение флагов/состояний

## Примеры

### ✅ Хорошо
```cpp
// Компактное представление RGB цвета
struct Color {
    unsigned int r : 8;  // 8 бит для красного (0-255)
    unsigned int g : 8;  // 8 бит для зеленого
    unsigned int b : 8;  // 8 бит для синего
};  // Всего 24 бита вместо 96

// Флаги файловых прав доступа
struct FilePermissions {
    unsigned int read    : 1;
    unsigned int write   : 1;
    unsigned int execute : 1;
};  // Занимает 1 байт вместо 3

// Работа с регистром железа
struct ControlRegister {
    unsigned int enable     : 1;
    unsigned int direction  : 2;
    unsigned int speed      : 3;
    unsigned int reserved   : 2;
};
```

### ❌ Плохо
```cpp
// Избыточное использование памяти
struct Color {
    unsigned int r;  // 32 бита для значения 0-255
    unsigned int g;  // еще 32 бита
    unsigned int b;  // и еще 32 бита
};  // Тратим 96 бит вместо 24

// Неправильное использование для больших чисел
struct BadDesign {
    int value : 31;  // Плохая идея для больших чисел
    // Лучше использовать обычный int
};

// Нарушение выравнивания без необходимости
struct Misaligned {
    char a;        // 1 байт
    int b : 3;     // Создаст паддинг
    char c;        // Снова паддинг
};
```

## Нюансы использования

### 1. Порядок битов
- Порядок битов зависит от компилятора/платформы
- Не полагайся на конкретный порядок битов
- Используй макросы для определения порядка если важно

### 2. Выравнивание и паддинг

- Структура всегда выравнивается по максимальному выравниванию полей
- Если есть поле 8 байт (например double), вся структура выравнивается по 8 байт
- Меньшие поля получают паддинг для соответствия максимальному выравниванию

```cpp
// Пример с 8-байтовым выравниванием
struct Example8 {
    char a;      // 1 байт + 7 байт паддинга
    double b;    // 8 байт
    int c;       // 4 байта + 4 байта паддинга
}; // Итого 24 байта

// Пример с 4-байтовым выравниванием
struct Example4 {
    char a;      // 1 байт + 3 байта паддинга
    int b;       // 4 байта
    int c;       // 4 байта
}; // Итого 12 байт

// Оптимальная группировка по размеру
struct Aligned {
    int a;           // 4 байта
    int b;           // 4 байта
    unsigned x : 4;  // Битовые поля вместе
    unsigned y : 4;
};
```

### 3. Побитовые операции

#### Применение
- Флаги состояний (вкл/выкл, права доступа, настройки)
- Упаковка данных (когда важна память)
- Работа с регистрами железа (драйверы, embedded)
- Хэш-функции и криптография
- Оптимизация производительности (быстрые математические операции)
- Сетевые протоколы (работа с заголовками пакетов)
- Сжатие данных

#### Основные операции с битами:
- `&` - AND (проверка бита)
- `|` - OR (установка бита)
- `^` - XOR (переключение бита)
- `~` - NOT (инверсия битов)
- `<<` - сдвиг влево (умножение на 2^n)
- `>>` - сдвиг вправо (деление на 2^n)

```cpp
struct Flags {
    unsigned int bits : 8;
};

void bitOperations() {
    Flags f{};
    
    // Установка бита
    f.bits |= (1 << 2);    // Установить 3-й бит
    
    // Проверка бита
    bool isSet = f.bits & (1 << 2);  // Проверить 3-й бит
    
    // Сброс бита
    f.bits &= ~(1 << 2);   // Сбросить 3-й бит
    
    // Переключение бита
    f.bits ^= (1 << 2);    // Инвертировать 3-й бит
    
    // Маски
    const unsigned int MASK = 0x3;    // Маска для младших 2 бит
    f.bits &= MASK;        // Оставить только младшие 2 бита
}
```

### 4. Доступ к битовым полям
- Нельзя взять адрес битового поля
- Операции могут быть медленнее обычных полей
- Атомарные операции не поддерживаются

```cpp
struct BitField {
    unsigned int flags : 4;
    unsigned int mode  : 4;
};

void examples() {
    BitField bf{};
    
    // ✅ Корректное использование
    bf.flags = 0x3;        // Прямая запись
    int value = bf.mode;   // Прямое чтение
    bf.flags |= 0x1;       // Побитовые операции работают

    // ❌ Не скомпилируется
    unsigned int* ptr = &bf.flags;  // Нельзя взять адрес
    std::atomic<unsigned int> atom{bf.flags};  // Нет атомарных операций
    
    // ⚠️ Возможны накладные расходы
    for(int i = 0; i < 1000; ++i) {
        bf.flags++;  // Каждая операция требует чтения/маскирования/записи
    }
}

### 4. Размер и паддинг
```cpp
struct Control {
    unsigned int flags : 16;
    unsigned int mode  : 16;
};  // Компилятор может добавить паддинг
```

### 5. Кроссплатформенность
- Размер битовых полей должен быть <= размера базового типа
- Используй фиксированные типы (uint32_t) для предсказуемости
- Проверяй размеры через static_assert
