- Логи — очень важный инструмент для анализа работы вашего приложения: они подскажут, если с приложением возникнет проблема, а вы сможете вовремя её устранить. 
## Хороший логер должен обладать двумя функциями:
- Добавлять временнУю метку к каждому логу.
- Изменять путь для записи логов. Это может быть стандартный вывод, файл или даже stringstream, чтобы тестировать логер.
- Попробуйте написать такой логер. Часть его описания и клиентский код, использующий логер, уже готовы.
- Скопируйте к себе описание логера и выполните задания:
- Определите конструктор класса Logger.
- Определите метод log, который принимает неограниченное число аргументов для вывода состояний разных переменных и удовлетворяет условиям:
- - Метод должен добавлять в начало сообщения дату и время записи в формате "[%Y-%m-%d %H:%M:%S]".
- - Метод должен разделять аргументы с помощью указанного в конструкторе сепаратора.
- - Лог должен заканчиваться символом конца строки.

```cpp
class Logger {
public:
    Logger(std::ostream &stream, const std::string &separator = ", ");
    // Здесь ваш код
};

int main() {
    Logger logger(std::cout, ", ");  // Разделитель между аргументами

    logger.log("User logged in", 42, 3.14, "Hello, world!");
    // Пример вывода:
    // [2023-10-05 12:34:56] User logged in, 42, 3.14, Hello, world!

    logger.log("Processing data", 100, 200, 300);
    // Пример вывода:
    // [2023-10-05 12:34:57] Processing data, 100, 200, 300

    return 0;
} 
```

- Сделайте метод log-вариабельным шаблоном.
- Перенаправляйте логирование в вариабельную функцию-хелпер, которая на каждом этапе своего вызова «отщипывает» от пакета один параметр и записывает его в стрим.
- Используйте рекурсию функции-хелпера, чтобы продолжать «отщипывать» параметры от пакета до тех пор, пока он не опустеет.
- Для формирования даты и времени логов используйте функции std::time, std::localtime, std::put_time.
- Используйте конструкцию sizeof..., чтобы узнать, пуст ли пакет параметров.
