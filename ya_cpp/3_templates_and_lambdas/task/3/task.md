Каждому разработчику хочется писать чистый безошибочный код. 

Для этого программа должна сигнализировать обо всех нештатных ситуациях и корректно обрабатывать эти сигналы — единообразно передавать значения ошибки для любых типов.

 Например, если функции с С-подобной сигнатурой возвращают -1, значит, произошла ошибка. 
 
 На нештатную ситуацию указывают и фабричные функции std::unique_ptr, которые возвращают nullptr. 
 
 Было бы здорово понимать, что функция возвращает ошибку, не погружаясь в её значение. Выглядеть это будет примерно так:

```cpp
int write_bytes(char *ptr, int size) {
    int result = 0;
    // do something
    if (error) {
        return error_value<int>;
    }
    return result;
}
// или

std::unique_ptr<SomeClass> create() {
    // do something
    if (error) {
        return error_value<std::unique_ptr<SomeClass>>;
    }
    // return good value
} 
```

Такой код подскажет, в каких случаях возвращается ошибка, а вам не придётся думать, какое значение она принимает. 

Скопируйте код ниже себе и реализуйте шаблон переменной error_value, который содержит ошибочное значение для каждого типа:

- Шаблон для типа int должен иметь значение -1.
- Шаблон для типов указателя или умных указателей - nullptr.

```cpp
template <typename T>
T error_value;

// Here is your code 
```

- Используйте полную специализацию для типа int, которая будет иметь значение -1.
- Используйте отдельные частичные специализации для типов указателей, std::unique_ptr и std::shared_ptr, которые будут иметь значения nullptr.
