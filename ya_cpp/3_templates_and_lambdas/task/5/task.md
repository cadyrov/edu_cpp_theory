Как бы тщательно мы ни оптимизировали код, некоторые функции выполняются долго, просто потому что объем вычислений большой или нужно обращаться с запросом к другим сервисам, которые могут долго отвечать. 

Например, вычисление n-го числа последовательности Фибоначчи уже при n = 50 может занимать десятки секунд. 

Дпугой пример: приложению банка нужно посчитать кредитный рейтинг клиента. Для этого оно по сети обращается в сервис, который вычисляет рейтинг. Работа с сетью всегда занимает намного больше времени, чем внутренний вызов процедуры.

А если нам понадобится второй раз посчитать кредитный рейтинг клиента? Конечно, мы не можем бесконечно оптимизировать вычисления, но можем предотвратить обработку запросов с одними и теми же данными. Для этого воспользуемся кешем, в котором по определённому ключу будем складывать результаты вычислений. В случае повторных данных ничего вычислять не будем, а будем возвращать сохранённый результат.

С помощью CRTP этот кеш можно сделать доступным для любых классов и подмешать в них функциональность кеша. Пользовательский код может выглядеть так:

```cpp
class Fibonacci : public Cache<Fibonacci> {
public:
    size_t Calculate(size_t n) {
        if (n <= 1)
            return n;
        return Calculate(n - 1) + Calculate(n - 2);
    }
};

class CreditScore : public Cache<CreditScore> {
public:
    double Calculate(const std::string &person_id) { return CalculateCreditScore(person_id); }
};

Fibonacci fib;
// в первый раз вычисляем
auto result2 = fib.compute(30);
// во второй раз достаём значение из кеша
auto result2 = fib.compute(30);

CreditScore cs;
// в первый раз вычисляем
auto result3 = cs.compute("123");
// во второй раз достаём значение из кеша
auto result4 = cs.compute("123"); 
```

Скопируйте код ниже и реализуйте класс Cache c единственным методом compute с помощью CRTP-техники. При этом помните:

Метод compute должен автоматически определять параметры входного и выходного значений метода Calculate у наследников.

Метод compute должен сохранять в ассоциативный контейнер результаты вычислений функций-потомков.

При попадании в метод аргумента, для которого уже вычислялась функция-наследник, Calculate должен возвращать результат из кеша, а не вычислять функцию повторно.

```cpp
? class Cache {
public:
    ? ? compute(?) ? { ? }
};
```  


- Для хранения кэша используйте статическую локальную переменную ассоциативного контейнера.
- Для получения универсального ключа кэша используйте std::hash.
- Чтобы получить тип возвращаемого значения функции-потомка Calculate, используйте оператор decltype.
