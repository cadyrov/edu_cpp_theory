# Задания по управлению ресурсами

## 1. Ручное управление памятью: malloc/free, new/delete

1. Выделите `int` через `new`, запишите значение и освободите через `delete`.
2. Выделите массив `int` через `new[]`, заполните и освободите через `delete[]`.
3. Напишите функцию, которая создает строку через `new std::string` и возвращает сырой указатель.
4. Исправьте вызов этой функции так, чтобы память точно освобождалась.
5. Выделите память через `malloc` и освободите через `free`.
6. Покажите, почему объект с конструктором нельзя корректно создать одним `malloc`.
7. Напишите пример неправильной пары `new[]` и `delete`.
8. Напишите класс, который логирует конструктор и деструктор, и проверьте вызовы при `new/delete`.
9. Напишите функцию, где при раннем `return` легко забыть `delete`.
10. Перепишите этот пример без ручного `delete`.

## 2. Проблемы использования new/delete

1. Напишите пример утечки памяти без `delete`.
2. Напишите пример утечки при раннем `return`.
3. Напишите пример утечки при исключении между `new` и `delete`.
4. Исправьте утечку через `try/catch` и ручной `delete`.
5. Покажите, почему такой ручной код сложно поддерживать.
6. Напишите функцию с двумя ресурсами, где второй `new` может бросить исключение.
7. Исправьте пример через локальные объекты стандартной библиотеки.
8. Напишите пример double delete.
9. Напишите пример dangling pointer после `delete`.
10. Сформулируйте правило: кто владеет указателем и кто обязан освобождать.

## 3. Эксклюзивное владение std::unique_ptr

1. Создайте `std::unique_ptr<int>` через `std::make_unique`.
2. Проверьте, что `unique_ptr` нельзя скопировать.
3. Передайте владение другому `unique_ptr` через `std::move`.
4. Проверьте состояние исходного `unique_ptr` после move.
5. Напишите функцию, которая принимает `std::unique_ptr<T>` по значению.
6. Напишите функцию, которая возвращает `std::unique_ptr<T>`.
7. Используйте `get()` для передачи сырого указателя без владения.
8. Используйте `reset()` для замены объекта.
9. Используйте `release()` и вручную освободите полученный сырой указатель.
10. Напишите `unique_ptr` с custom deleter для `FILE*`.

## 4. Современная реализация PImpl

1. Создайте класс `Widget` с публичным методом `print`.
2. Вынесите приватные поля в структуру `Widget::Impl`.
3. Храните `Impl` через `std::unique_ptr<Impl>`.
4. Оставьте в заголовке только forward declaration `Impl`.
5. Определите деструктор `Widget` в `.cpp`.
6. Запретите копирование `Widget`.
7. Разрешите перемещение `Widget`.
8. Добавьте `const`-метод и проверьте доступ к `Impl`.
9. Добавьте метод, меняющий состояние внутри `Impl`.
10. Объясните, какие зависимости исчезли из заголовка.

## 5. Move-семантика и обмен состояниями

1. Напишите класс с copy constructor и выведите сообщение при копировании.
2. Добавьте move constructor и выведите сообщение при перемещении.
3. Проверьте вызов copy для lvalue.
4. Проверьте вызов move через `std::move`.
5. Напишите move assignment.
6. После move оставьте исходный объект в валидном состоянии.
7. Реализуйте `swap` для класса с ресурсом.
8. Используйте copy-and-swap для assignment.
9. Напишите функцию, возвращающую локальный объект, и проверьте NRVO.
10. Уберите лишний `std::move` из `return` локального объекта.

## 6. Правила нуля, трех и пяти

1. Напишите класс по правилу нуля на базе `std::string`.
2. Напишите класс с сырым указателем и деструктором.
3. Добавьте copy constructor для глубокого копирования.
4. Добавьте copy assignment.
5. Добавьте move constructor.
6. Добавьте move assignment.
7. Запретите копирование для move-only класса.
8. Перепишите класс с сырым указателем на `std::unique_ptr`.
9. Сравните количество ручного кода до и после `unique_ptr`.
10. Напишите `static_assert`, проверяющий копируемость и перемещаемость типа.

## 7. RAII и ScopeGuard

1. Напишите класс, который печатает сообщение в конструкторе и деструкторе.
2. Создайте объект в блоке и проверьте момент вызова деструктора.
3. Напишите RAII-обертку для `FILE*`.
4. Запретите копирование этой обертки.
5. Добавьте move constructor для передачи владения.
6. Напишите простой `ScopeGuard`, вызывающий лямбду в деструкторе.
7. Используйте `ScopeGuard` для отката изменения переменной.
8. Добавьте метод `dismiss`, отменяющий действие guard.
9. Проверьте работу `ScopeGuard` при исключении.
10. Используйте RAII для пары операций `open/close` или `lock/unlock`.

## 8. Готовые состояния функции

10. Напишите небольшой pipeline из нескольких функций с состоянием.

## 9. jemalloc и jeprof

1. Соберите небольшую программу с большим количеством аллокаций.
2. Добавьте флаг `-g` при сборке.
3. Добавьте `-fno-omit-frame-pointer`.
4. Соберите программу с подключением `jemalloc`.
5. Запустите программу с `MALLOC_CONF=prof:true`.
6. Получите heap profile при завершении программы.
7. Посмотрите профиль через `jeprof --text`.
8. Найдите функцию с наибольшим вкладом в аллокации.
9. Сравните два профиля через `--base`.
10. Сформулируйте, чем heap profiling отличается от leak checker.
