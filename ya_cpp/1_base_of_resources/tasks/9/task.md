Попробуйте применить библиотеку WIL для создания и закрытия дочерних процессов в Windows (Windows child processes). 

Перепишите код, который создаёт процесс и немедленно его закрывает, с помощью класса-обёртки из библиотеки WIL.

В правильном решении должна остаться только одна строка - создание RAII-объекта.

Скопируйте код ниже к себе и выполните задания:

В документации найдите объект wil::unique_, который правильно работает с HANDLE.

Перепишите код с помощью выбранного класса. Вызывать функцию закрытия HANDLE в итоговом решении не нужно.

Важно: Библиотека WIL предоставляет несколько классов-обёрток для работы с HANDLE, но вам нужен тот, который работает непосредственно с HANDLE, полученными при создании Process.

```cpp
HANDLE hProcess = OpenProcess(PROCESS_ALL_ACCESS, FALSE, GetCurrentProcessId());
if (hProcess) {
    CloseHandle(hProcess);
} 
```

 - Поиском по слову process найдите, в названии каких типов в документации есть слово «process», а затем выберите самый подходящий класс для хранения объекта типа HANDLE, полученного при вызове OpenProcess.
